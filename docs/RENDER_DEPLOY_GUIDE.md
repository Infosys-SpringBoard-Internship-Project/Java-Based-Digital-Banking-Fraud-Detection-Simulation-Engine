# Render Deployment Guide (Step-by-Step)

This guide deploys FraudShield as two Docker services on Render:
- `fraudshield-ml` (Flask model API)
- `fraudshield-app` (Spring Boot + dashboard)

Use this guide exactly in order.

## 1) Before you start

You need:
- A GitHub account
- A Render account
- A Supabase (or PostgreSQL) database URL, username, and password
- This project pushed to GitHub with `render.yaml` in root

## 2) Push code to GitHub

From project root (`fraud-project-source/`):

```bash
git init
git add .
git commit -m "Prepare project for Render deployment"
git branch -M main
git remote add origin https://github.com/<your-username>/<your-repo>.git
git push -u origin main
```

Notes:
- `.gitignore` already excludes local/runtime files (venv, logs, target, node_modules, env files).
- `license.txt` is included and can be published.

## 3) Verify required files exist in repo root

- `render.yaml`
- `Dockerfile`
- `ml/Dockerfile`
- `pom.xml`
- `src/`
- `ml/`

## 4) Create services in Render using Blueprint

1. Open Render dashboard.
2. Click **New +**.
3. Click **Blueprint**.
4. Connect your GitHub repo.
5. Select branch (`main`).
6. Render reads `render.yaml` and shows two services.
7. Click **Apply** / **Create Blueprint**.

Expected services:
- `fraudshield-ml`
- `fraudshield-app`

## 5) Set environment variables

### 5.1 On `fraudshield-ml`
Usually none required for basic run.

Optional (only if you customize paths):
- `PORT` (Render sets this automatically)

### 5.2 On `fraudshield-app`
Set these in Render service -> **Environment**:

Required:
- `SPRING_DATASOURCE_URL=jdbc:postgresql://db.<project-ref>.supabase.co:5432/postgres?sslmode=require`
- `SPRING_DATASOURCE_USERNAME=postgres`
- `SPRING_DATASOURCE_PASSWORD=<your-db-password>`
- `ML_API_URL=https://fraudshield-ml.onrender.com/predict` (or your ML service URL)
- `ML_HEALTH_URL=https://fraudshield-ml.onrender.com/health`

Recommended:
- `ML_AUTOTRAIN_ENABLED=false` (already set in `render.yaml` for safer free-tier behavior)
- `MAIL_SENDER=<your-mail-id>`
- `MAIL_PASSWORD=<app-password>`

Important:
- Use the real URL generated by Render for the ML service.
- Do not set local URLs (`127.0.0.1`) on Render.

## 6) Deploy order (important)

1. Deploy `fraudshield-ml` first.
2. Open ML service URL + `/health` and verify response is healthy.
3. Then deploy `fraudshield-app`.

Why:
- App checks ML health and calls ML `/predict` endpoint.

## 7) Verify deployment after build

### ML checks
- `https://<ml-service>.onrender.com/health` should return running status.

### App checks
- `https://<app-service>.onrender.com/pages/index.html`
- `https://<app-service>.onrender.com/pages/admin-login.html`
- `https://<app-service>.onrender.com/system/health`

## 8) First-time product setup

1. Open app login page.
2. If no admin exists, register initial superadmin (`/auth/register` flow via UI).
3. Login and validate dashboard pages load.
4. Test one transaction validation from UI.

## 9) Common Render issues and fixes

### Error: app starts but DB connection fails
- Recheck `SPRING_DATASOURCE_URL`, username, password.
- Ensure Supabase DB allows external connections.
- Ensure `sslmode=require` is present.

### Error: app is up but ML says down
- Check `ML_API_URL` and `ML_HEALTH_URL` are using deployed ML URL.
- Verify ML service is healthy and not suspended.

### Error: unauthorized/redirect loops
- Ensure browser allows cookies for session.
- Clear old cookies and login again.

### Error: slow startup / timeout
- Free tier cold starts are expected.
- Retry after first boot, then monitor logs.

## 10) Updating deployment after code changes

1. Push changes to GitHub `main` branch.
2. Render auto-deploys (`autoDeploy: true`).
3. Verify both service builds and health endpoints.

## 11) Recommended production hardening checklist

- Use stronger DB user policy.
- Keep `ML_AUTOTRAIN_ENABLED=false` in hosted environment unless controlled.
- Configure email only with app passwords.
- Rotate secrets if leaked.
- Add custom domain and HTTPS policy (Render supports TLS by default).
- Add backup/restore plan for Postgres.
